# AI 知识学习闯关平台 — 需求分析文档

## 一、项目概述

### 1.1 项目定位

**一句话描述**：用户输入任意知识内容，AI 自动生成闯关题目，通过答题互动的方式帮助用户高效学习和记忆。

**产品形态**：Web 端优先（后期扩展小程序 / App）

**Slogan 建议**：
- "学一遍不如闯一关"
- "输入知识，闯关掌握"
- "AI 出题，越学越上瘾"

### 1.2 目标用户画像

| 维度 | 描述 |
|------|------|
| **核心用户** | 18-35 岁 C 端个人学习者（职场人士、终身学习者） |
| **使用场景** | 碎片化学习、知识巩固、考前复习、内容创作者自测 |
| **用户痛点** | 学了记不住、传统学习方式枯燥、缺乏即时反馈和成就感 |
| **用户特征** | 有一定自主学习意愿，喜欢游戏化体验，愿意为高效学习付费 |

### 1.3 核心价值主张

1. **降低学习门槛**：输入任意内容即可开始学习，无需手动整理笔记
2. **提升学习效果**：通过主动回忆（Active Recall）和间隔重复（Spaced Repetition）原理强化记忆
3. **增加学习趣味**：游戏化闯关机制让学习不再枯燥
4. **可视化成果**：学习报告和分享机制让进步可见

### 2.2 主要竞品分析

#### 竞品一：Kahoot!

| 维度 | 分析 |
|------|------|
| **定位** | 课堂互动答题平台，偏向教学场景 |
| **核心功能** | 教师创建题目 → 学生实时答题 → 排行榜竞争 |
| **用户规模** | 全球 9 亿+ 累计用户，覆盖 200+ 国家 |
| **盈利模式** | 免费基础版 + 付费高级版（$3-$9/月） |
| **AI 能力** | 2024 年引入 AI 辅助出题，但功能较浅 |
| **优势** | 品牌知名度高、课堂场景成熟、实时互动体验好 |
| **劣势** | 需要教师主导创建内容、学习深度不足、非自学场景 |
| **与本项目差异** | Kahoot 需要人工出题，本项目 AI 自动生成；Kahoot 偏课堂，本项目偏自学 |

#### 竞品二：Quizlet

| 维度 | 分析 |
|------|------|
| **定位** | 数字记忆卡片工具，偏向记忆场景 |
| **核心功能** | 闪卡、学习模式、测试模式、匹配游戏 |
| **用户规模** | 全球 6000 万+ 月活用户 |
| **盈利模式** | 免费基础版 + Quizlet Plus（$7.99/月） |
| **AI 能力** | Q-Chat（AI 助手）、AI 生成闪卡、自适应学习 |
| **优势** | 用户基数大、学习模式多样、社区内容丰富 |
| **劣势** | 闯关感弱、题目类型有限、知识获取依赖用户输入 |
| **与本项目差异** | Quizlet 偏记忆卡片，本项目偏知识理解；Quizlet 需手动创建，本项目全自动 |

#### 竞品三：Duolingo

| 维度 | 分析 |
|------|------|
| **定位** | 语言学习游戏化平台 |
| **核心功能** | 关卡制课程、连续打卡、排行榜、成就系统 |
| **用户规模** | 全球 1 亿+ 月活用户 |
| **盈利模式** | 免费 + Super Duolingo（$12.99/月）+ 广告 |
| **AI 能力** | AI 对话练习、自适应难度、个性化推荐 |
| **优势** | 游戏化设计标杆、用户粘性极高、商业模式成熟 |
| **劣势** | 仅限语言学习、内容制作成本高 |
| **与本项目差异** | Duolingo 专注语言，本项目覆盖全学科；Duolingo 内容人工制作，本项目 AI 生成 |

#### 竞品四：国内相关产品

| 产品 | 特点 | 与本项目差异 |
|------|------|-------------|
| **猿题库** | K12 题库刷题，偏向应试 | 题库固定，非 AI 生成，偏应试 |
| **作业帮** | 拍照搜题 + 在线课程 | 偏工具属性，非闯关学习 |
| **百词斩** | 图片联想背单词 | 仅限英语单词，场景单一 |
| **得到** | 知识付费音频课程 | 被动学习为主，缺乏互动 |

### 2.3 竞品差异化总结

| 差异化维度 | 现有竞品 | 本项目优势 |
|-----------|---------|-----------|
| **知识来源** | 用户手动创建或固定题库 | AI 自动从任意内容生成 |
| **出题方式** | 人工出题或简单模板 | AI 智能出题，理解语义 |
| **学习场景** | 单一场景（语言/考试/课堂） | 全学科、全内容类型 |
| **学习深度** | 记忆为主 | 理解 + 应用 + 分析（布鲁姆分类法） |
| **即时反馈** | 仅显示对错 | 对错 + 知识讲解 + 拓展 |

**结论：本项目在"AI 自动生成 + 闯关互动 + 全学科覆盖"这个交叉点上存在明确的差异化空间。**

## 三、市场分析

### 3.1 市场规模

| 市场 | 规模 | 增长趋势 |
|------|------|---------|
| **全球 EdTech 市场** | 2025 年约 4000 亿美元 | 年增长率 16%+ |
| **AI 教育市场** | 2025 年约 200 亿美元 | 年增长率 35%+ |
| **中国在线教育市场** | 2025 年约 6000 亿人民币 | AI 驱动新增长 |
| **游戏化学习市场** | 2025 年约 30 亿美元 | 年增长率 25%+ |

### 3.2 市场趋势

1. **AI 原生教育产品崛起**：2024-2026 年，AI 从"辅助工具"变为"核心引擎"，大量 AI-native 教育产品涌现
2. **个性化学习需求爆发**：用户不再满足于标准化内容，期望根据自身水平定制学习路径
3. **碎片化学习成为主流**：移动端、短时高频的学习方式取代长时间集中学习
4. **游戏化渗透率提升**：Duolingo 验证了游戏化模式的商业价值，更多领域开始采用
5. **知识焦虑持续存在**：终身学习理念普及，成人学习市场持续扩大

| 机会维度 | 评分 | 说明 |

| 用户需求 | ⭐⭐⭐⭐ | 学习效率和趣味性需求真实存在 |
| 竞争格局 | ⭐⭐⭐⭐ | 直接竞品少，差异化空间大 |

## 五、功能需求分析

### 5.1 功能全景

本项目功能分为 **核心功能 MVP** 和 **扩展功能** 两部分。

#### 核心功能 MVP（按用户操作流程设计）

| 步骤 | 功能 | 描述 |
|------|------|------|
| 1 | 知识输入 | 用户自由输入想学的知识（一句话 / 一段话 / 文档 / 视频 / 网页） |
| 2 | 知识获取 | AI 自动从全网或特定信息源获取知识（网络搜索、解析文档/视频/网页） |
| 3 | 智能出题 | AI 自动生成交互式闯关题目、选项和答案，通过提问引导增加学习趣味性 |
| 4 | 答题闯关 | 用户自由答题闯关，每道题选择后立刻给出正确答案和知识讲解（答错也有讲解） |
| 5 | 学习报告 | 通关后生成知识总结和复盘报告，用户可分享给好友 |
| 6 | 后台分析 | 用户可在后台查看所学知识和答题情况，便于日后复盘，可生成报告 |

#### 扩展功能（后续迭代）

| 功能 | 描述 | 应用场景 |
|------|------|---------|
| RAG 知识库 | 允许输入知识库来生成题目 | 企业培训、特定题库准备、考试复习 |
| AI 生图 | 应用到题目和选项中，根据知识类型选择是否生成图片 | 刷英语单词、视觉记忆 |
| VIP 充值 | 高级功能付费解锁 | 盈利 |
| 多人对战 PK | 实时 1v1 或多人答题对战 | 社交裂变、竞争激励 |
| 排行榜 | 全局/好友/小组排行 | 竞争激励 |
| 数字人 | AI 数字人角色出题/讲解 | 趣味性、个性化 |
| 语音读题 | TTS 语音朗读题目 | 通勤、运动等场景 |

### 5.2 用户核心流程

```
用户输入知识内容
       │
       ▼
AI 解析和获取知识 ──────── 解析失败 → 提示用户重新输入
       │
       ▼
AI 生成闯关题目
       │
       ▼
用户开始答题闯关
       │
       ├── 答对 → 显示正确反馈 + 知识拓展 → 下一题
       │
       └── 答错 → 显示错误原因 + 知识讲解 → 下一题
              │
              ▼
       全部通关
              │
              ▼
       生成学习报告 ──────── 分享给好友
              │
              ▼
       保存到个人中心 ──────── 后台分析 / 后续复盘
```

## 六、亮点思路补充

基于竞品调研和市场分析，以下是额外的创新思路，可在 MVP 之后逐步加入：

### 6.1 自适应难度系统

**思路**：根据用户答题表现动态调整题目难度。

```
答对 3 题连续 → 难度 +1
答错 2 题连续 → 难度 -1
最终得分 = 基础分 × 难度系数
```

**价值**：避免太简单无聊或太难放弃，保持用户在"心流"状态。

### 6.2 知识薄弱点智能推荐

**思路**：分析用户错题分布，识别薄弱知识点，推荐针对性复习。

```
用户在"过拟合"相关题目错误率高
  → 推荐：过拟合专题闯关
  → 推荐：相关论文/文章阅读
```

**价值**：从"被动答题"升级为"主动学习"，提升学习效果。

### 6.3 社交学习机制

**思路**：
- **学习小组**：创建学习小组，组内成员可以比拼成绩
- **挑战好友**：把闯关链接发给好友，两人做同一套题比拼
- **知识 PK**：实时 1v1 答题对战

**价值**：社交裂变 + 竞争激励，提升用户粘性和传播。

### 6.4 学习打卡与连续激励

**思路**：
- 每日学习打卡，连续打卡获得奖励
- 连续 7 天解锁成就徽章
- 连续 30 天获得 VIP 体验

**价值**：借鉴 Duolingo 的连续打卡机制，培养用户习惯。

### 6.5 AI 学习助手对话

**思路**：在答题过程中，用户可以随时向 AI 提问。

```
用户：这道题我不太理解，能再解释一下吗？
AI：当然！过拟合可以这样理解...
用户：能举个生活中的例子吗？
AI：想象你在准备考试...
```

**价值**：从"单向出题"升级为"双向互动"，模拟真人辅导体验。

### 6.6 多模态学习

**思路**：
- **语音读题**：适合通勤、运动等场景
- **图片辅助**：AI 生成配图帮助理解抽象概念
- **视频解析**：输入视频链接，提取关键帧和字幕生成题目

**价值**：覆盖更多学习场景，提升可访问性。

### 6.7 知识地图与学习路径

**思路**：将知识组织成树状或图状结构，用户可以沿着学习路径逐步深入。

```
机器学习
├── 基础概念
│   ├── 监督学习 ← 当前
│   ├── 无监督学习
│   └── 强化学习
├── 算法
│   ├── 线性回归
│   ├── 决策树
│   └── 神经网络
└── 实践
    ├── 数据预处理
    ├── 模型评估
    └── 部署上线
```

**价值**：让碎片化学习变得系统化，用户能看到自己的学习全景。

### 7.3 AI 出题 Prompt 设计示例

```markdown
你是一个专业的教育出题专家。请根据以下知识内容，生成 {count} 道高质量的选择题。

## 知识内容
{knowledge_content}

## 要求
1. 题目类型：单选题
2. 每题 4 个选项（A/B/C/D），只有 1 个正确答案
3. 难度层次：
   - 40% 基础题（概念记忆、事实回忆）
   - 40% 中等题（理解应用、举例说明）
   - 20% 困难题（分析推理、对比判断）
4. 干扰项要有迷惑性，不能太明显错误
5. 每题附带详细的知识讲解（100-200字）

## 输出格式（JSON）
{
  "questions": [
    {
      "id": 1,
      "question": "题目内容",
      "options": {
        "A": "选项A",
        "B": "选项B",
        "C": "选项C",
        "D": "选项D"
      },
      "answer": "B",
      "explanation": "知识讲解内容",
      "difficulty": "medium",
      "knowledge_point": "相关知识点"
    }
  ]
}
```
