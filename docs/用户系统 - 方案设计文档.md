## 一、用户系统概述

### 1.1 功能范围

用户系统是 AI 知识闯关平台的核心模块，提供以下功能：

| 功能模块 | 说明 | 优先级 |
|---------|------|--------|
| 用户注册/登录 | 邮箱密码注册登录 + GitHub OAuth 登录 | P0 |
| 个人中心 | 用户信息展示、学习统计、连续学习天数 | P0 |
| 历史记录 | 答题历史列表、筛选、分页 | P0 |
| 错题本 | 错题列表、移除错题、重新练习 | P0 |

### 1.2 技术选型

| 组件 | 技术方案 | 说明 |
|------|---------|------|
| 认证框架 | Sa-Token 1.38+ | 轻量级、API 简洁、支持 OAuth2 |
| 密码加密 | BCrypt | Spring Security 内置，安全可靠 |
| 第三方登录 | GitHub OAuth 2.0 | 开发者友好，无需企业资质 |
| 前端状态管理 | Pinia | Vue 3 官方推荐 |

CREATE TABLE IF NOT EXISTS t_user (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,

    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY uk_email (email),
    UNIQUE KEY uk_auth (auth_type, auth_id)

### 3.2 认证流程设计

#### 3.2.1 邮箱注册/登录流程

```
用户提交邮箱+密码
       │
       ▼
   参数校验
       │
       ├── 注册：检查邮箱是否已注册 → 加密密码 → 创建用户
       │
       └── 登录：查找用户 → 验证密码
       │
       ▼
   Sa-Token 登录：StpUtil.login(userId)
       │
       ▼
   返回用户信息 + Token（自动写入 Cookie）
```

#### 3.2.2 GitHub OAuth 登录流程

```
用户点击"GitHub 登录"
       │
       ▼
前端请求 GET /api/v1/auth/github
       │
       ▼
后端返回 GitHub 授权 URL
       │
       ▼
前端跳转到 GitHub 授权页
       │
       ▼
用户授权后 GitHub 重定向到回调地址
GET /api/v1/auth/github/callback?code=xxx
       │
       ▼
后端用 code 换取 access_token
       │
       ▼
用 access_token 获取用户信息
       │
       ▼
查询/创建本地用户（uk_auth 唯一索引保证幂等）
       │
       ▼
Sa-Token 登录：StpUtil.login(userId)
       │
       ▼
重定向到前端 /oauth/callback?code=success
       │
       ▼
前端获取用户信息，跳转首页
```

### 3.4 连续学习天数算法

```java
public void updateStudyStats(Long userId, int correctCount, int totalCount) {
    User user = userMapper.selectById(userId);
    LocalDate today = LocalDate.now();

    // 更新答题统计
    user.setTotalQuizzes(user.getTotalQuizzes() + 1);
    user.setTotalCorrect(user.getTotalCorrect() + correctCount);
    user.setTotalQuestions(user.getTotalQuestions() + totalCount);

    // 更新连续学习天数
    if (user.getLastStudyDate() == null) {
        // 首次学习
        user.setStreakDays(1);
    } else if (user.getLastStudyDate().equals(today.minusDays(1))) {
        // 连续学习
        user.setStreakDays(user.getStreakDays() + 1);
    } else if (!user.getLastStudyDate().equals(today)) {
        // 中断后重新开始
        user.setStreakDays(1);
    }
    // 同一天学习则不更新连续天数

    user.setLastStudyDate(today);
    userMapper.updateById(user);
}
```

### 5.3 用户状态管理

```typescript
// stores/user.ts
export const useUserStore = defineStore('user', () => {
  const userInfo = ref<UserInfo | null>(null)
  const isLoggedIn = ref(false)

  async function fetchUserInfo() {
    try {
      const data = await getUserInfo()
      userInfo.value = data
      isLoggedIn.value = true
    } catch {
      userInfo.value = null
      isLoggedIn.value = false
    }
  }

  function setUser(info: UserInfo) {
    userInfo.value = info
    isLoggedIn.value = true
  }

  function clearUser() {
    userInfo.value = null
    isLoggedIn.value = false
  }

  return { userInfo, isLoggedIn, fetchUserInfo, setUser, clearUser }
})
```

## 六、GitHub OAuth 配置指南

### 6.1 创建 GitHub OAuth App

1. 登录 GitHub → 右上角头像 → **Settings**
2. 左侧菜单 → **Developer settings** → **OAuth Apps** → **New OAuth App**
3. 填写表单：
   - **Application name**: `AI 知识闯关`
   - **Homepage URL**: `http://localhost:5173`（开发环境）
   - **Authorization callback URL**: `http://localhost:8080/api/v1/auth/github/callback`
4. 点击 **Register application**
5. 获取 **Client ID**
6. 点击 **Generate a new client secret** → 复制保存

### 6.2 配置文件

**application-local.yml**（不提交 Git）：
```yaml
github:
  client-id: Ov23liyRB9sKr9eMRz0j
  client-secret: 38a20dbd302ae1d7821cd3707e6737facbd92fcc
  redirect-uri: http://localhost:8080/api/v1/auth/github/callback
```

## 七、安全设计

### 7.1 密码安全

- 使用 BCrypt 加密算法，每次加密结果不同
- 密码长度不少于 6 位
- 不在日志中输出密码

### 7.2 Token 安全

- Sa-Token 配置：
  - `timeout: 604800`（7 天有效期）
  - `active-timeout: 86400`（24 小时无操作过期）
  - `is-concurrent: true`（允许并发登录）
  - `is-share: false`（不共享 Token）

### 7.3 接口安全

- 需要登录的接口通过 Sa-Token 拦截器验证
- 401 错误自动跳转登录页
- CORS 配置允许跨域请求

### 7.4 敏感数据

- GitHub Client Secret 存储在 `application-local.yml`，不提交 Git
- 密码 BCrypt 加密后存储，无法逆向
