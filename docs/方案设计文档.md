## 一、技术选型与依据

### 1.1 整体技术栈

| 层级 | 技术选型 | 版本 | 选型依据 |
|------|---------|------|---------|
| **前端框架** | Vue 3 | 3.4+ | 响应式、组合式 API、中文生态好 |
| **构建工具** | Vite | 5.x | 开发体验极快，HMR 秒级刷新 |
| **UI 组件库** | Element Plus | 2.x | 成熟稳定，组件覆盖全，适合后台+表单场景 |

| **图表库** | ECharts | 5.x | 学习报告中的数据可视化 |

| **ORM 框架** | MyBatis Plus | 3.5+ | 单表零 SQL、代码生成器、分页插件 |
| **认证授权** | Sa-Token | 1.38+ | 轻量级、API 简洁、支持 OAuth2 第三方登录 |
| **数据库** | MySQL | 8.0+ | 成熟稳定，社区资源丰富 |
| **缓存** | Redis | 7.x | 会话管理、排行榜、热点数据缓存 |
| **AI 模型** | DeepSeek | deepseek-chat | 性价比极高（约 ¥1/百万 token），中文能力强，出题质量好 |

| **API 文档** | Knife4j (Swagger) | 4.x | 在线 API 文档，方便前后端联调 |

### 1.2 AI 模型选型深度对比

| 维度 | DeepSeek | 通义千问 | Claude | GPT-4 |
|------|----------|---------|--------|-------|
| **中文出题质量** | ★★★★★ | ★★★★ | ★★★★ | ★★★★ |
| **JSON 输出稳定性** | ★★★★★ | ★★★★ | ★★★★★ | ★★★★ |
| **性价比** | ★★★★★ | ★★★★ | ★★★ | ★★ |
| **国内访问** | 直连 | 直连 | 需代理 | 需代理 |
| **Token 价格** | ¥1/百万 | ¥2/百万 | ¥15/百万 | ¥60/百万 |

**结论**：DeepSeek 作为主力模型，后期可扩展多模型支持。

```sql
    INDEX idx_session_id (session_id),
```

## 四、API 接口设计

### 4.1 接口规范

- **基础路径**：`/api/v1`
- **认证方式**：Sa-Token，通过 Header `satoken: xxx` 传递 token
- **统一响应格式**：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

- **错误码规范**：

| 错误码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未登录 |
| 403 | 无权限 |
| 500 | 服务器内部错误 |
| 1001 | AI 生成题目失败 |
| 1002 | 知识内容过短/过长 |
| 1003 | 答题会话不存在或已结束 |

### 4.2 接口清单

## 五、核心模块详细设计

### 5.1 模块一：AI 出题服务

这是整个系统的**核心模块**，直接决定产品质量。

#### 5.1.1 出题流程

```
用户输入知识内容
       │
       ▼
内容预处理（清洗、截断、格式化）
       │
       ▼
构建 Prompt（注入知识内容 + 出题要求）
       │
       ▼
调用 DeepSeek API（流式/非流式）
       │
       ▼
解析 JSON 响应 → 容错处理
       │
       ├── 成功 → 存入数据库 → 返回前端
       │
       └── 失败 → 重试（最多3次）→ 降级处理
```

#### 5.1.2 Prompt 工程设计

**核心出题 Prompt**（`prompts/generate-questions.txt`）：

```
你是一个专业的教育出题专家。请根据以下知识内容，生成 {count} 道高质量的选择题。

## 知识内容
{knowledge_content}
```

#### 5.1.3 DeepSeek API 调用封装

```java
@Service
public class AiService {

    @Value("${ai.deepseek.api-key}")
    private String apiKey;

    @Value("${ai.deepseek.base-url}")
    private String baseUrl;

    private final RestTemplate restTemplate = new RestTemplate();

    public QuestionGenerationResult generateQuestions(String content, int count) {
        String prompt = buildPrompt(content, count);

        HttpHeaders headers = new HttpHeaders();
        headers.set("Authorization", "Bearer " + apiKey);
        headers.setContentType(MediaType.APPLICATION_JSON);

        Map<String, Object> body = Map.of(
            "model", "deepseek-chat",
            "messages", List.of(
                Map.of("role", "system", "content", "你是一个专业的教育出题专家，只输出JSON格式的题目。"),
                Map.of("role", "user", "content", prompt)
            ),
            "temperature", 0.7,
            "max_tokens", 4096,
            "response_format", Map.of("type", "json_object")
        );

        HttpEntity<Map<String, Object>> request = new HttpEntity<>(body, headers);

        // 调用 API（带重试）
        for (int i = 0; i < 3; i++) {
            try {
                ResponseEntity<Map> response = restTemplate.exchange(
                    baseUrl + "/v1/chat/completions",
                    HttpMethod.POST,
                    request,
                    Map.class
                );
                String jsonContent = extractContent(response.getBody());
                return parseQuestions(jsonContent);
            } catch (Exception e) {
                if (i == 2) throw new BizException(1001, "AI 生成题目失败，请重试");
            }
        }
        throw new BizException(1001, "AI 生成题目失败");
    }
}
```

#### 5.1.4 容错与降级策略

| 场景 | 处理方式 |
|------|---------|
| JSON 解析失败 | 使用正则提取 JSON 块，忽略前后多余文字 |
| 题目数量不足 | 补发请求生成剩余题目 |
| API 超时 | 3 次重试，间隔 2s/4s/8s（指数退避） |

#### 5.2.3 得分计算算法

```java
public int calculateScore(int correctCount, int totalCount,
                          int difficultyBonus, int streakBonus) {
    // 基础分 = 正确率 × 70
    double baseScore = (double) correctCount / totalCount * 70;

    // 难度加分 = 难度系数 × 15
    // easy=0, medium=1, hard=2
    double difficultyScore = difficultyBonus * 5;

    // 连对加分 = 最大连对数 / 总题数 × 15
    double streakScore = (double) streakBonus / totalCount * 15;

    return (int) Math.min(100, baseScore + difficultyScore + streakScore);
}
```

| 风险 | 概率 | 影响 | 应对方案 |
|------|------|------|---------|
| DeepSeek API 不稳定 | 中 | 高 | 预留通义千问作为备用模型，统一封装切换 |
| AI 输出 JSON 格式异常 | 中 | 中 | 多层容错：严格解析 → 正则提取 → 重新请求 |
| AI 出题质量差 | 中 | 高 | Prompt 持续迭代 + 用户反馈机制 + A/B 测试 |
| API 成本超预期 | 低 | 中 | 缓存相同内容的出题结果 + 限流 + 成本监控 |
| 并发性能瓶颈 | 低 | 中 | 异步出题 + 线程池隔离 + 水平扩展 |
